This is a curated list of the user-level commands you are most
likely to want.  Internal / less common commands are omitted; you
can still discover them with =M-x bmkx-= =TAB=.

* Jumping

** =bmkx-jump= minibuffer

When =consult= is loaded, =bmkx-jump='s minibuffer also accepts:

| Key                | Effect                                                         |
|--------------------+----------------------------------------------------------------|
| =,= then a char    | Type narrow (see =bmkx-jump-narrow=): =,f= file, =,d= Dired, =,i= Info, =,o= Org heading, =,w= Web, ... |
| =M-t=              | Add a tag filter (completing-read over every distinct tag).    |
| =M-D=              | Pop the most recently added filter.                            |
| =M-T=              | Clear all active filters.                                      |
| Typing a tag name  | Filters by tag (hidden suffix is searchable).                  |

User options:

- =bmkx-jump-sort-by= --- =mru= (default), =visits=, or =alpha=.
- =bmkx-jump-group-by= --- =nil= (default), =type=, or =tag=.
- =bmkx-jump-narrow= --- type-narrow key map; see the docstring.
- =bmkx-bookmark-location-function= --- function called with a
  bookmark record; returns a string to substitute for marginalia's
  =- no file -= marker in the annotation, or nil to leave it.
  Default returns the bookmark's =location= property, so URL / EWW /
  W3M / browse-URL bookmarks show their URL.
- =bmkx-jump-candidate-format-function= --- function that builds the
  visible candidate row.  Called with the bookmark record and the
  narrow alist; must return a string carrying the
  =bmkx-bookmark-name= and =consult--type= text properties.  Use
  the helper =bmkx-jump-candidate-type-char= to compute the narrow
  char.  Default =bmkx-jump-candidate-default= shows the bookmark
  name and appends a hidden =#tag1 #tag2= suffix.
