* Introduction

/Bookmark+/ is a substantial extension of GNU Emacs's built-in
=bookmark.el=.  It keeps every concept from vanilla bookmarks ---
named, persistent placeholders into files, buffers, Info nodes, and
so on --- but makes bookmarks useful as persistent Emacs records,
not just saved positions.

Bookmark+ enlarges vanilla bookmarks in three directions:

1. *Richer bookmarks.*  Bookmarks can carry tags, annotations,
   region information, visit counts, last-access timestamps, and
   arbitrary user data.  They can be highlighted in the buffer they
   point to, and they remember where they were even when the file
   has been edited.
2. *More kinds of bookmarks.*  In addition to bookmarks that "go" to
   a location, Bookmark+ ships over 70 bookmark types: Dired
   buffers, Info nodes, Gnus articles, EWW/W3M pages, =man= pages,
   image files, URLs, desktops, keyboard macros, sets of variables,
   snippets of text, sequences of other bookmarks, whole bookmark
   files, and more.  You can also create bookmarks that have no
   "destination" at all and are simply named persistent records.
3. *More to do with bookmarks.*  The =*Bookmark List*= buffer is
   significantly enhanced.  You can mark, sort, filter, omit,
   search, query-replace, tag, retag, edit, clone, highlight,
   describe, and copy/move bookmarks across multiple bookmark
   files.  You can save bookmark-list views, define your own jump
   commands and sort orders, and cycle through arbitrary sets of
   bookmarks ("the navigation list").

The package is the work of Drew Adams.  Its canonical online
documentation lives on the Emacs Wiki:
https://www.emacswiki.org/emacs/BookmarkPlus.  An in-tree, more
detailed manual is in =bookmark+-doc.el=, also accessible from
inside Emacs (see [[#documentation-help][Built-in documentation and help]]).

This README is a user guide and reference.  It starts with the
problems Bookmark+ solves and the commands most users need first,
then continues into the larger feature reference.

* Why Bookmark+?
:PROPERTIES:
:CUSTOM_ID: why-bookmark-plus
:END:

Vanilla Emacs bookmarks are useful, but they are deliberately
small: they mostly name places and let you jump back to them.
Bookmark+ is for workflows where bookmarks become part of how you
organize and restore Emacs state.

Use Bookmark+ when you want to:

- Keep many bookmarks understandable with tags, annotations,
  filtering, sorting, and saved views.
- Bookmark things other than simple file positions: Dired buffers,
  Info nodes, Gnus articles, EWW/W3M pages, images, URLs, desktops,
  keyboard macros, variable sets, snippets, bookmark files, and
  sequences of other bookmarks.
- Return to a working context, not just a location: a Dired buffer
  with markings, a saved desktop, a restored narrowing, or a list of
  bookmarks you can cycle through.
- Act on bookmarks in bulk: mark them, tag them, search their
  targets, query-replace through them, copy or move them to another
  bookmark file, or define a command from the current view.
- Treat bookmarks as lightweight persistent records, including
  non-invokable records that do not jump anywhere.

The center of this workflow is the =*Bookmark List*= buffer.  It is
not just a list of names; it is the place where you inspect,
filter, mark, edit, sort, tag, save, and operate on bookmarks.

* Quick start
:PROPERTIES:
:CUSTOM_ID: quick-start
:END:

After loading Bookmark+, these commands cover the basic workflow:

| Command                                  | Key                         | What it does                                      |
|------------------------------------------+-----------------------------+---------------------------------------------------|
| =bmkp-bookmark-set-confirm-overwrite=      | =C-x r m=, =C-x x m=          | Set or update a bookmark at point.                |
| =bookmark-jump=                            | =C-x j j=, =C-x r b=          | Jump to any bookmark.                             |
| =bookmark-bmenu-list=                      | =C-x x e=, =C-x r l=          | Open =*Bookmark List*=.                           |
| =bmkp-add-tags=                            | =C-x x t + b=                 | Add tags to a bookmark.                           |
| =bmkp-this-file/buffer-bmenu-list=         | =C-x x ,=                     | Show bookmarks for the current file or buffer.    |
| =bmkp-next-bookmark-this-file/buffer-repeat= | =C-x x n=, =C-x x down=       | Cycle forward through bookmarks here.             |
| =bmkp-switch-bookmark-file-create=         | =C-x x L=                     | Switch to another bookmark file.                  |

Suggested first session:

1. Create a bookmark with =bmkp-bookmark-set-confirm-overwrite=
   (=C-x r m=).
2. Open =*Bookmark List*= with =bookmark-bmenu-list= (=C-x x e=).
3. Add tags with =bmkp-add-tags= (=T += in =*Bookmark List*=, or
   =C-x x t + b= globally).
4. Jump with =bookmark-jump= (=C-x j j=) or from the list with
   =bookmark-bmenu-this-window= (=RET=).
5. Use =bmkp-bmenu-show-all= (=.=), sorting keys such as =s s=,
   and filtering keys such as =T S= to shape the list.

* Installation

:CUSTOM_ID: installation

** From source

1. Place all =bookmark+*.el= files in a directory on your =load-path=.
2. Byte-compile them after loading the source files in this order
   (Bookmark+'s macros must be present before any user of them is
   compiled):
   #+begin_src emacs-lisp
   (load-file "bookmark+-mac.el")     ; first, so macros are defined
   (byte-compile-file "bookmark+-mac.el")
   (byte-compile-file "bookmark+-1.el")
   (byte-compile-file "bookmark+-bmu.el")
   (byte-compile-file "bookmark+-lit.el")  ; optional
   (byte-compile-file "bookmark+-key.el")
   (byte-compile-file "bookmark+.el")
   #+end_src
3. Add this to your init file:
   #+begin_src emacs-lisp
   (require 'bookmark+)
   #+end_src

That single =require= loads every required Bookmark+ library.  If
you do not want highlighting, simply do not put =bookmark+-lit.el=
on your =load-path=.

** Load order and =bookmark-default-file=

The value of =bookmark-default-file= is consulted when Bookmark+
loads, to initialize =bmkp-last-as-first-bookmark-file= and
=bmkp-current-bookmark-file=.  If you set =bookmark-default-file=
in your init file, *set it before loading Bookmark+*.  In general,
set =bookmark-default-file= once, then use
=bmkp-switch-bookmark-file-create= (=C-x x L=) to switch as needed.

** Caveats when migrating from old Bookmark+

In 2010 the package's prefix changed from =bookmarkp-= to =bmkp-=.
If your init file, bookmark file, =*Bookmark List*= state file, or
=*Bookmark List*= commands file pre-date that change, replace all
=bookmarkp-= occurrences with =bmkp-= before loading Bookmark+.  Do
this manually under =emacs -Q= or with auto-save turned off; see
the warning at the top of =bookmark+-doc.el=.

** Back up your bookmark file

Bookmark+ writes backups when it saves your bookmark file
(vanilla Emacs does not, due to bug #12507).  For safety, set
=bookmark-version-control= to =t= so numbered backups are kept, and
=delete-old-versions= to =t= to suppress the prompt at save time.

* Concepts: list, file, display
:PROPERTIES:
:CUSTOM_ID: concepts
:END:

Three different things are commonly confused.  Understanding them
is the single most important thing for using Bookmark+ effectively.

1. *The bookmark list.*  The in-memory set of bookmarks.  It is
   the value of internal variable =bookmark-alist=.  Adding,
   renaming, deleting, or jumping to a bookmark updates this list.
2. *The bookmark file.*  An on-disk persistent record of a set of
   bookmarks.  Its default location is =bookmark-default-file=
   (=~/.emacs.bmk=).  Multiple bookmark files may exist; only one
   is /current/ at a time.
3. *The bookmark-list display.*  Buffer =*Bookmark List*= (also
   known as the bookmark "menu list").  It is a snapshot view of
   the bookmark list, rendered with markings, sort order, filters,
   and so on.

These three can be out of sync.  The list rules during an Emacs
session.  The file rules across sessions (it is all that survives
when Emacs exits).  In the display, =bmkp-bmenu-refresh-menu-list=
(=g=) refreshes from the list; with a prefix argument (=C-u g=), it
reloads from the file.

#+begin_quote
Tip.  Use =bookmark-bmenu-list= (=C-x x e= or =C-x r l=) to open
=*Bookmark List*= as your "home page" for navigating files,
projects, and notes.  Bookmark+ restores the buffer's previous
state (sort order, filtering, marks) each time you open it.
#+end_quote

* Prefix keys and keymaps
:PROPERTIES:
:CUSTOM_ID: prefix-keys
:END:

Bookmark+ allocates dedicated prefix keys for bookmarking,
separating them from the shared =C-x r= register prefix.

| Keymap                          | Default key | Purpose                                |
|---------------------------------+-------------+----------------------------------------|
| =bookmark-map=                    | =C-x x=      | General bookmark commands.              |
| =bmkp-set-map=                    | =C-x x c=    | Create (set) new bookmarks.             |
| =bmkp-tags-map=                   | =C-x x t=    | Tag commands.                           |
| =bmkp-annotate-map=               | =C-x x a=    | Annotation commands.                    |
| =bmkp-jump-map=                   | =C-x j=      | Jump (visit) bookmarks --- same window.  |
| =bmkp-jump-other-window-map=      | =C-x 4 j=    | Jump --- other window.                   |
| Vanilla other-frame jump          | =C-x 5 j=    | Jump --- other frame.                    |

The vanilla =C-x r m= / =C-x r b= / =C-x r l= bindings still work.

The prefixes are themselves customizable through three list-valued
options:

- =bmkp-bookmark-map-prefix-keys= (default ="C-x x"=)
- =bmkp-jump-map-prefix-keys= (default ="C-x j"=)
- =bmkp-jump-other-window-map-prefix-keys= (default ="C-x 4 j"=)

Because they are lists, you can bind a keymap under several
prefixes at once, e.g. both =C-x x= and =<f9>=.  Re-binding is done
by customizing these options --- a setter function rebinds the keys
for you.

Inside the =*Bookmark List*= buffer, single-letter keys take over:
=bookmark-bmenu-this-window= (=J=) mirrors =bmkp-jump-map= (=C-x
j=), and the other-window jump prefix (=j=) mirrors
=bmkp-jump-other-window-map= (=C-x 4 j=).  See
[[#key-binding-tables][Key-binding tables]] for the full inventory.

** =bmkp-bookmark-set-confirm-overwrite= --- vanilla set, with extras

=bmkp-bookmark-set-confirm-overwrite= (=C-x r m=, with aliases
=C-x x c m= and =C-x x m=) is an enhanced replacement for
=bookmark-set=:

- Completion candidates default to bookmarks in the current
  buffer (or all bookmarks if there are none) so you can easily
  update a nearby bookmark.  A numeric prefix argument expands
  completion to all bookmarks.
- Overwriting prompts for confirmation.
- Existing tags and annotations are preserved when you re-set a
  bookmark (see =bmkp-properties-to-keep=).

** Bookmarking without visiting

You can bookmark targets without first opening them:

- *A file or URL at point.*  =bmkp-file-target-set= (=C-x x c f=)
  and =bmkp-url-target-set= (=C-x x c u=) read a file or URL with
  the text at point as the default.
- *Compilation/grep/occur hits.*  In =*compilation*=, =*grep*=,
  =*Occur*=, and =*Multi-Occur*= buffers:
  - =bmkp-compilation-target-set= or =bmkp-occur-target-set=
    (=C-c C-b=) bookmarks the target of the hit at point.
  - =bmkp-compilation-target-set-all= or =bmkp-occur-target-set-all=
    (=C-c C-M-b=) bookmarks the target of every hit.
  - =bmkp-occur-create-autonamed-bookmarks= (=C-c C-M-B= in
    =*Occur*=) creates an autonamed bookmark per hit.
- *Marked files in Dired* (requires =dired+.el=):
  - =diredp-do-bookmark= (=M-b=) bookmarks each marked file as an
    autofile.
  - The Dired+ =C-M-B= binding bookmarks each into a bookmark file
    you specify.
  - The Dired+ =C-M-b= binding does the same and also creates a
    bookmark-file bookmark for the resulting file.

  =C-u C-u= / =C-u C-u C-u= / =C-u C-u C-u C-u= as the prefix
  argument widens the scope from "marked" to "all files (no
  directories)" / "all files and dirs except . and .." / "all files
  and all directories".

** Jumping with completion

=bookmark-jump= (=C-x j j=) is the generic bookmark jump command;
completion offers every bookmark.  =bookmark-jump-other-window=
(=C-x 4 j j=) is its other-window twin.  Inside =*Bookmark List*=,
=bookmark-bmenu-this-window= (=J=) and the other-window jump prefix
(=j=) jump same- / other-window.

For specific types, type-specific jump commands offer only
bookmarks of that type as completion candidates.  They live on the
=bmkp-jump-map= (default =C-x j=) and =bmkp-jump-other-window-map=
(default =C-x 4 j=).  Examples:

** =find-file=-style jumping

=bmkp-find-file= (=C-x j C-f=) and =bmkp-find-file-other-window=
(=C-x 4 j C-f=) act like =find-file=, but obey
=bmkp-default-handlers-for-file-types= --- so HTML files open in
=browse-url=, Word docs open in =w32-browser=, and so on (see
[[#bookmark-types][Bookmark types]] below).  With a prefix argument they also create
an autofile bookmark for the file.

There is also a family of =bmkp-find-file-...-tags= commands
(under =C-x j t C-f ...=) that filter by tags while reading a file
name with =read-file-name= (so you can browse directories).

** Mode-specific =j= keys

Several modes bind =j= to the corresponding type-specific jump:

|----------------------------+------------------------------------------|
| Info                       | Info bookmark.                            |
| Gnus summary               | Gnus bookmark.                            |
| =man= / =woman=             | =man=-page bookmark.                       |
| W3M (mode and minor mode)  | W3M bookmark.                             |
| EWW                        | EWW bookmark.                             |

| Dired (=C-j=)               | Dired bookmark whose dir = current dir.   |

** Tag-based jumping

Tag-aware jump commands have key sequences that combine three
ideas: which bookmark family (file, autofile, this-dir-file,
find-file), how many tags must match (all = =*=, some/any = =+=),
and whether the match is literal or against a regexp (=%= = regexp).

| Command                                             | Key                    | Meaning                                  |
|-----------------------------------------------------+------------------------+------------------------------------------|
| =bmkp-all-tags-jump=                                  | =C-x j t *=             | Bookmark with all given tags.            |
| =bmkp-some-tags-jump=                                 | =C-x j t +=             | Bookmark with at least one given tag.    |
| =bmkp-all-tags-regexp-jump=                           | =C-x j t % *=           | Bookmark whose every tag matches regexp. |
| =bmkp-some-tags-regexp-jump=                          | =C-x j t % +=           | Bookmark with some tag matching regexp.  |
| =bmkp-file-{all,some}-tags-{regexp-}jump=             | =C-x j t f * / +/...=   | File or directory bookmark.              |
| =bmkp-autofile-{all,some}-tags-{regexp-}jump=         | =C-x j t a * / + /...=  | Autofile bookmark.                       |
| =bmkp-file-this-dir-{all,some}-tags-{regexp-}jump=    | =C-x j t . * /...=      | File bookmark in =default-directory=.     |
| =bmkp-find-file-{all,some}-tags-{regexp-}jump=        | =C-x j t C-f * /...=    | =read-file-name= version (browses dirs). |

Each command has an =-other-window= twin under =C-x 4 j t ...=.

** Cycling: the navigation list

Bookmark+ lets you /cycle/ through a set of bookmarks --- like the
"favorites" buttons on a TV remote.  The set being cycled is the
"navigation list" (navlist).

Define the navlist explicitly:

- =bmkp-choose-navlist-of-type= (=C-x x :=) --- all bookmarks of a
  given type.
- =bmkp-choose-navlist-from-bookmark-list= (=C-x x B=) ---
  bookmarks corresponding to a bookmark-list bookmark, or the
  current =*Bookmark List*= state.

If you do not define it, the first cycling key takes a snapshot of
the current bookmarks (or of the bookmarks for the current
file/buffer, for the file/buffer-specific cycling keys).

Cycle through the navlist (with wraparound):

| Command                                      | Forward key                 | Backward key                |
|----------------------------------------------+-----------------------------+-----------------------------|
| =bmkp-{next,previous}-bookmark-repeat=         | =C-x x f / C-f / right=       | =C-x x b / C-b / left=       |
| =bmkp-{next,previous}-bookmark-w32-repeat=     | =C-x x next= (PageDown)       | =C-x x prior= (PageUp)       |

Cycle bookmarks in the current file/buffer (dynamic set, not a
snapshot):

| Command                                                   | Forward key             | Backward key        |
|-----------------------------------------------------------+-------------------------+---------------------|
| =bmkp-{next,previous}-bookmark-this-file/buffer-repeat=     | =C-x x n / C-n / down=   | =C-x x p / C-p / up= |
| =bmkp-{next,previous}-lighted-this-buffer-repeat=           | =C-x x C-down=           | =C-x x C-up=         |

After =C-x x= you can also rotate the mouse wheel to cycle (Emacs
23.3 or later).

Show =*Bookmark List*= filtered to the navigation list with
=bmkp-navlist-bmenu-list= (=C-x x N=).

* Bookmark types
:PROPERTIES:
:CUSTOM_ID: bookmark-types
:END:

A bookmark "type" is really just a /handler/: a function that
"jumps" to (= acts on) the bookmark.  Anything can be a bookmark
type.  Bookmark+ ships handlers for around 70 of them.

Run =M-x bmkp-types-alist= to list every type registered in your
session.  =bmkp-jump-to-type= (=C-x j :=) lets you pick a type
interactively.

Predicates of the form =bmkp-TYPE-bookmark-p= recognize each type.

** Location / file bookmarks

- *File* / *local file* / *remote file*.  Jumping visits the file.
  =bmkp-jump-{file,local-file,remote-file}=.
- *Directory*.  Like a file bookmark, but the destination is a
  directory.
- *Non-file (buffer)*.  Like a file bookmark for a buffer that is
  not visiting a file --- the buffer must exist when you jump.
- *Region*.  A bookmark with =front-context-string=,
  =rear-context-string=, plus an =end-position= property.  Jumping
  activates the region.  See =bmkp-use-region= for the activation
  policy (=t=, =cycling-too=, or =nil=).  Use
  =bmkp-region-jump-narrow-indirect-other-window= (=C-x 4 j R=) to
  narrow the region in a cloned indirect buffer (needs
  =narrow-indirect.el=).

** Type-aware file actions

=bmkp-default-handlers-for-file-types= is the "MIME map" of
Bookmark+.  Each entry pairs a regexp matching a file name with a
shell command (string) or Lisp function (symbol or lambda) that
will be invoked on the file:

#+begin_src emacs-lisp
;; Example value:
(("\\.ps$"    . "gsview32.exe")
 ("\\.html?$" . browse-url)
 ("\\.doc$"   . w32-browser))
#+end_src

These mappings affect:

- =bmkp-bookmark-a-file= (=C-x x c a=) and =bmkp-autofile-set=,
  which save the action as the bookmark's =file-handler= property.
- =bmkp-find-file= / =bmkp-find-file-other-window= (=C-x j C-f= /
  =C-x 4 j C-f=).

If =bmkp-guess-default-handler-for-file-flag= is non-nil and no
mapping matches, Bookmark+ consults =dired-guess-shell-alist-default=
and (Emacs 23+) mailcap entries.

On MS-Windows, =M-RET= / =M-mouse-2= / =M-o= in =*Bookmark List*=
open bookmarks via Windows file associations (requires
=w32-browser.el=).

** Autofile bookmarks

An /autofile/ is a bookmark whose name is the non-directory part
of its file name.  =bmkp-autofile-set= (=C-x x c a=) creates one
for a file.  The
=bmkp-propertize-bookmark-names-flag= option (non-nil by default on
Emacs 21+) lets you have multiple autofiles with the same name in
different directories.  Combined with tags this gives you a kind
of file-system-wide metadata layer.  See also: =bmkp-autofile-filecache=
(advise =file-cache-add-file= to also create autofiles),
=bmkp-autofile-access-invokes-bookmark-flag= (make =find-file=
also "visit" the autofile, updating its visit count).

** Dired and Dired-tree bookmarks

A Dired bookmark records:

- the Dired directory,
- the =ls= switches,
- which files are marked,
- which subdirectories are inserted,
- which subdirectories are hidden.

Jumping reproduces the Dired buffer with the same markings/inserted
subdirs/hidden subdirs.  A /Dired tree bookmark/ is a sequence
bookmark whose components are Dired bookmarks for a directory and
its descendants.  =diredp-do-bookmark-dirs-recursive= (from
=dired+.el=) creates one for the current Dired buffer.

** Info bookmarks

=bmkp-info-jump= (=C-x j i= or =j= in Info mode).  Mode
=bmkp-info-auto-bookmark-mode= automatically creates / updates an
Info bookmark every time you visit a node.
=bmkp-info-auto-type= picks =update-only= (the default) or
=create-or-replace=; toggle with =bmkp-toggle-info-auto-type=.

** Gnus, =man=, =woman=, W3M, EWW

- =bmkp-gnus-jump= bookmark of a Gnus article (group, article,
  message id).
- =bmkp-man-jump= for =man= or =woman= pages.
- =bmkp-w3m-jump=, =bmkp-eww-jump= for W3M and EWW URLs.
- EWW can be configured so that visiting a page records a
  bookmark, via =bmkp-set-eww-bookmark-here= and
  =bmkp-eww-replace-keys-flag= (rebinds vanilla EWW bookmark keys
  to Bookmark+'s).

** Image bookmarks

When you bookmark an image-file target, Emacs's image handler is
used automatically.  =bmkp-image-jump= (=C-x j M-i=) jumps to one;
the description (=C-h RET=) shows the full EXIF metadata (Emacs
22+ with =exiftool=).

** URL bookmarks

=bmkp-url-target-set= (=C-x x c u=) and =bmkp-url-jump= (=C-x j u=).
The handler is typically =browse-url= or =w32-browser=.

** Desktop bookmarks

=bmkp-set-desktop-bookmark= (=C-x x K= or =C-x x c K= or =C-x r K=)
captures a desktop (from =desktop.el=).  Jumping restores the
saved desktop.  Options:

- =bmkp-desktop-no-save-vars= --- variables to exclude.
- =bmkp-desktop-jump-save-before-flag= --- save current desktop
  before jumping if it was reached by jumping.
- =bmkp-desktop-save-as-last= --- add to =kill-emacs-hook= to
  remember the last jumped-to desktop.

** Sequence (composite) bookmarks

A sequence bookmark invokes each member bookmark in turn.
=bmkp-set-sequence-bookmark= (=C-x x c s=) prompts for a sequence
name and the member bookmark names.  =bmkp-bmenu-make-sequence-from-marked=
turns the marked bookmarks (in their current order) into a
sequence.  Members may themselves be sequence bookmarks.

** Function (and keyboard-macro) bookmarks

=bmkp-make-function-bookmark= (=C-x x c F=) records a function
symbol or a lambda expression.  With a prefix argument, it
captures =last-kbd-macro=, giving you a persistent way to replay
a keyboard macro.  =bmkp-wrap-bookmark-with-last-kbd-macro=
(=C-x x c C-k=) builds a sequence bookmark that first invokes
another bookmark and then plays the macro.

A prefix argument given when jumping is passed to the function (or
counts macro repetitions).

** Variable-list bookmarks

=bmkp-set-variable-list-bookmark= records a set of variables and
their current values.  =bmkp-jump-variable-list= (=C-x j v=)
restores them in the bookmark's buffer.  =bmkp-set-izones-bookmark=
(needs =zones.el=) records a buffer's =zz-izones= (ring of
narrowings) for later restoration.

** Snippet bookmarks

=bmkp-set-snippet-bookmark= (=C-x x M-w= or =C-x x c M-w=) saves
the active region's text under a bookmark name.  At any time,
in any Emacs session, =bmkp-snippet-to-kill-ring= (=C-x j M-w= /
=C-x 4 j M-w=) yanks the saved snippet to the kill ring.

** Bookmark-list bookmarks

A /bookmark-list bookmark/ records the state of =*Bookmark List*=
(sort order, filter, regexp pattern, title, omit list).  Jumping
to it restores that state.  =bmkp-bmenu-define-command= (=C-c C-c=
in =*Bookmark List*=) defines a command that does the same thing,
without consuming a bookmark slot.  =bmkp-bmenu-define-full-snapshot-command=
(=C-c C-S-c=, aka =C-c C-C=) captures /everything/ including marks
--- larger but complete.

** Bookmark-file bookmarks

Created by =bmkp-set-bookmark-file-bookmark= (=C-x x y= /
=C-x x c y=).  "Jumping" to one /loads/ the referenced bookmark
file.  With =C-u=, jumping /switches/ to it.

** Other built-in types

- *Icicles search-hits* bookmarks --- save the current
  Icicles-search candidate set; restore with =bmkp-jump-icicle-search-hits=.
- *Lighter (highlight) override* --- per-bookmark style/face/condition
  for highlighting (see [[#highlighting][Highlighting]]).
- *Non-invokable* --- handler is =ignore=.  Useful for using
  bookmarks just as named persistent records (e.g. Isearch filter
  predicate suites).  Shown with the =bmkp-no-jump= face.

** Defining your own types

Define a handler function and set =bookmark-make-record-function=
(or pass a handler via your own bookmark setter).  To register a
new type so =bmkp-jump-to-type= recognizes it, provide a function
=bmkp-TYPE-alist-only= and call =(bmkp-define-history-variables)=.
The macro =bmkp-define-type-from-handler= (in =bookmark+-mac.el=)
does both, plus it generates a predicate =bmkp-TYPE-bookmark-p=
and a "show only" command =bmkp-bmenu-show-only-TYPE-bookmarks=
via =bmkp-define-show-only-command=.

* Tags
:PROPERTIES:
:CUSTOM_ID: tags
:END:

A bookmark tag is a string (or a string + Lisp value) attached to
a bookmark.  Tags are persistent; multiple bookmarks can share the
same tag; one bookmark can have many.  Tags define /sets/ of
bookmarks ("blue", "vacation/2024/photos", "todo"...) and most
Bookmark+ commands can operate on those sets.

** Adding / removing tags

| Command                                             | Global key              | Bmenu key        | Meaning                                |
|-----------------------------------------------------+-------------------------+------------------+----------------------------------------|
| =bmkp-add-tags=                                      | =C-x x t + b=            | =T +=             | Add tags to a bookmark.                |
| =bmkp-remove-tags=                                   | =C-x x t - b=            | =T -=             | Remove tags from a bookmark.           |
| =bmkp-remove-all-tags=                               | =C-x x t 0=              | =T 0=             | Remove all tags from a bookmark.       |
| =bmkp-tag-a-file= (alias =bmkp-autofile-add-tags=)   | =C-x x t + a=            | (none)           | Tag a file (autofile creation).        |
| =bmkp-untag-a-file=                                  | =C-x x t - a=            | (none)           | Remove tags from a file (autofile).    |
| =bmkp-copy-tags=                                     | =C-x x t c= / =M-w=      | =T c= / =T M-w=   | Copy this bookmark's tags.             |
| =bmkp-paste-add-tags=                                | =C-x x t p= / =C-x x t C-y= | =T p=          | Paste copied tags (additive).          |
| =bmkp-paste-replace-tags=                            | =C-x x t q=              | =T q=             | Paste copied tags (replace).           |
| =bmkp-rename-tag=                                    | =C-x x t r=              | =T r=             | Rename a tag everywhere.               |
| =bmkp-remove-tags-from-all=                          | =C-x x t d=              | =T d=             | Delete a tag from all bookmarks.       |
| =bmkp-edit-tags=                                     | =C-x x t e=              | =T e=             | Edit a bookmark's tags in a buffer.    |
| =bmkp-list-all-tags=                                 | =C-x x t l=              | =T l=             | List every defined tag.                |
| =bmkp-set-tag-value=                                 | =C-x x t v=              | =T v=             | Give a tag a value.                    |
| =bmkp-set-tag-value-for-navlist=                     | =C-x x t V=              | =T V=             | Same for every bookmark in the navlist.|

You can press =RET RET= at the tag prompt to accept the empty set
(many commands treat that as "any tag at all").  Use =C-q C-j= to
include a newline in a tag.

Tag-name completion candidates are controlled by
=bmkp-tags-for-completion= (default: =current= = use the tags from
the current bookmark list; you can also list specific tags or
specific bookmark files).

If =bmkp-prompt-for-tags-flag= is non-nil, you are prompted for
tags when you create or update any bookmark.

** Marking by tag in =*Bookmark List*=

The tag-based mark and unmark commands all live under =T m= / =T u=:

| Command                                           | Key          | Marks/unmarks bookmarks tagged ... |
|---------------------------------------------------+--------------+------------------------------------|
| =bmkp-bmenu-mark-bookmarks-tagged-all=              | =T m *=       | with all of the given tags.         |
| =bmkp-bmenu-mark-bookmarks-tagged-some=             | =T m +=       | with some of the given tags.        |
| =bmkp-bmenu-mark-bookmarks-tagged-not-all=          | =T m ~ *=     | not with all of them.               |
| =bmkp-bmenu-mark-bookmarks-tagged-none=             | =T m ~ +=     | with none of them.                  |
| =bmkp-bmenu-mark-bookmarks-tagged-regexp=           | =T m %=       | with a tag matching a regexp.       |
| =bmkp-bmenu-unmark-bookmarks-tagged-...=            | =T u *= ...=  | unmark variants.                    |
| =bmkp-bmenu-show-only-tagged-bookmarks=             | =T S=         | show only tagged bookmarks.         |
| =bmkp-bmenu-show-only-untagged-bookmarks=           | =T U=         | show only untagged bookmarks.       |
| =bmkp-define-tags-sort-command=                     | =T s=         | define a sort command keyed on tags.|

Prefix argument flips "some" ↔ "not all" and "all" ↔ "none".

** Tag values

Use =bmkp-set-tag-value= (=T v= in =*Bookmark List*=, or
=C-x x t v= globally) to give a tag any readable Lisp value:
number, string, list, etc.  A tag with a value is internally
=(NAME . VALUE)=.  Most Bookmark+ commands ignore values; you can
use them from your own Lisp.

A single predefined tag with special behaviour: =bmkp-jump=.  If
its value is a function, that function is called just after
=bookmark-after-jump-hook= runs for the tagged bookmark.

Tag names prefixed with =bmkp-= are reserved.

** Pseudo-hierarchical tags

A simple convention --- tag names that look like paths
(=vacation/2024/winter/photos=) --- gives you a tree-shaped
organization without any special syntax.  Use completion (=TAB=)
on partial prefixes to descend the "tree".

* Annotations
:PROPERTIES:
:CUSTOM_ID: annotations
:END:

An annotation is free-text metadata attached to a bookmark.

- =bmkp-annotate-bookmark= (=C-x x a a=) prompts for a bookmark and
  opens an annotation edit buffer.  =bookmark-edit-annotation= is
  similar; without a prefix arg it lists only annotated bookmarks
  as candidates.
- =bookmark-show-annotation= (=C-x x a s=) shows an annotation in
  read-only mode.  =C-x C-q= toggles to edit mode.
- =bookmark-show-all-annotations= (=C-x x a S=).
- =bmkp-annotate-bookmark-this-file/buffer= (=C-x x a b=) and
  =bmkp-annotate-all-bookmarks-this-file/buffer= (=C-x x a B=).

In the edit buffer:

- =bookmark-send-edited-annotation= saves (=C-c C-c= in most
  annotation buffers; =C-c C-M-c= when Org mode owns =C-c C-c=).
- =bookmark-kill-annotation= (=C-x C-k=) discards changes.
- =C-x C-q= toggles buffer modes.
- The buffer's major mode is controlled by
  =bmkp-annotation-modes-inherit-from= (default: =org-mode=).
- =bookmark-automatically-show-annotations= --- if non-nil, jumping
  to a bookmark pops up its annotation (=edit= to open in edit
  mode, anything else for read-only).

** External annotations ("annotation forwards")

You can keep the inline annotation short and "forward" to a file,
URL, or another bookmark.  Put one of these as the first non-blank
line of the annotation:

#+begin_example
bmkp-annot-url: "FILE"      ; opens with find-file-other-window
bmkp-annot-url: "URL"       ; opens with browse-url
bmkp-annot-url: "BOOKMARK"  ; jumps to the named bookmark
#+end_example

When you "show" the annotation, Bookmark+ follows the forward.
When you /edit/ it (=a e=), you still see the literal text and any
extra lines after the forward.

** In =*Bookmark List*=

| Command                                              | Key           | Meaning                                  |
|------------------------------------------------------+---------------+------------------------------------------|
| =bookmark-bmenu-show-annotation=                       | =a a=          | Show this line's annotation.             |
| =bookmark-bmenu-edit-annotation=                       | =a e=          | Edit this line's annotation.             |
| =bookmark-bmenu-show-all-annotations=                  | =a A=          | Show all annotations.                    |
| =bmkp-bmenu-edit-annotations-for-marked=               | =a >=          | Edit annotations for all marked bookmarks. |
| =bmkp-bmenu-show-this-annotation+move-{down,up}=       | =M-down/M-up=  | Move + show annotation.                  |

Bookmarks with annotations are flagged with =a= in the display.

* The =*Bookmark List*= buffer
:PROPERTIES:
:CUSTOM_ID: bmenu
:END:

Open it with =bookmark-bmenu-list= (=C-x x e= or =C-x r l=).  It
is the heart of Bookmark+'s interactive workflow: filtering,
marking, sorting, tagging, highlighting, editing --- everything
happens here.

** Header

The header shows the current bookmark file and a /title/ that
describes the current filter (e.g. ="File and Directory Bookmarks"=
or ="All Bookmarks"=).  The mode line shows counts: marked (=>=),
flagged for deletion (=D=), tagged (=t=), temporary (=X=),
annotated (=a=), modified-unsaved (=*=), plus the sort order.
Where a counter is split, =N/M= means "N up to and including the
current line, of M total".

Per-line flags:

| Mark | Meaning                                                |
|------+--------------------------------------------------------|
| =>=    | Marked (a-la Dired).                                   |
| =D=    | Flagged for deletion.                                  |
| =a=    | Has an annotation.                                     |
| =t=    | Has at least one tag.                                  |
| =X=    | Temporary --- will not be saved.                        |
| =*=    | Modified since last save of the bookmark file.         |
| pink one-character highlight | Has a per-bookmark =lighting= override. |

** Marking

| Command                                      | Key           | Action                                 |
|----------------------------------------------+---------------+----------------------------------------|
| =bookmark-bmenu-mark=                          | =m=            | Mark this bookmark.                    |
| =bookmark-bmenu-unmark=                        | =u=            | Unmark this bookmark.                  |
| =bmkp-bmenu-unmark-all=                        | =M-DEL=, =U=   | Unmark all (or only =>=, or only =D=). |
| =bmkp-bmenu-toggle-marks=                      | =t=            | Swap marked / unmarked.                |
| =bmkp-bmenu-mark-all=                          | =M-m=          | Mark all.                              |
| =bmkp-bmenu-regexp-mark=                       | =% m=          | Mark by regexp on the bookmark line.   |
| =bmkp-bmenu-flag-for-deletion=                 | =d=, =k=       | Flag for deletion (=D=).               |
| =bmkp-bmenu-flag-for-deletion-backwards=       | =C-d=          | Same, moving up.                       |
| =bmkp-bmenu-delete-marked=                     | =D=            | Delete all flagged/marked bookmarks.   |

The =F M= / =I M= / =A M= ... family marks specific kinds (see
"filtering by type" below).

** Filtering

Three independent kinds of filtering:

1. *By type* --- show only bookmarks of one kind.  Keys end in =S=
   (show).

   #+begin_example
   F S  - file/dir bookmarks       A S  - autofile bookmarks
   B S  - non-file bookmarks       Y S  - bookmark-file bookmarks
   R S  - region bookmarks         Z S  - bookmark-list bookmarks
   I S  - Info bookmarks           K S  - desktop bookmarks
   G S  - Gnus bookmarks           V S  - variable-list bookmarks
   M S  - man-page bookmarks       N S  - non-invokable bookmarks
   M-I M-S - image bookmarks       Q S  - function bookmarks
   W3 S - W3M bookmarks            i S  - Icicles search-hits
   WE S - EWW bookmarks            w S  - snippet bookmarks
   X S  - temporary bookmarks      O S  - orphaned local file bookmarks
   #S   - autonamed bookmarks      M-d M-s - Dired bookmarks
   T S  - tagged bookmarks         T U  - untagged bookmarks
   H S  - highlighted bookmarks    - S   - omitted bookmarks
   #+end_example

   For each, the corresponding =M= key (e.g. =F M=, =I M=) /marks/
   bookmarks of that type instead of showing only them.

   The current type filter is remembered when you save a
   bookmark-list bookmark or use =C-c C-c=.

2. *Incremental matching* --- type =P PREFIX REGEXP=.  As you
   type, only matching bookmarks remain visible.  Hit a
   non-self-inserting key (e.g. =RET=) to commit.

   #+begin_example
   P B  - filter by bookmark name (incremental regexp)
   P F  - filter by file name
   P A  - filter by annotation
   P T  - filter by tag
   #+end_example

3. *By mark* --- toggle "show only marked" (=>=) or "show only
   unmarked" (=<=).  =.= shows all.

Use =M-t= to toggle showing file names (locations) for each
bookmark.

** Omitting (vs. filtering)

/Omitting/ is a stronger form of hiding: omitted bookmarks are
hidden from every view except the dedicated omit view.

| Command                                      | Key    | Meaning                              |
|----------------------------------------------+--------+--------------------------------------|
| =bmkp-bmenu-omit/unomit-marked=                | =- >=   | Move marked bookmarks to/from omit.   |
| =bmkp-bmenu-show-only-omitted-bookmarks=       | =- S=   | Show only the omitted ones.           |
| =bmkp-unomit-all=                              | =- U=   | Un-omit everything (can be called outside =*Bookmark List*=). |

Component bookmarks of a new sequence bookmark are omitted by default.

* Multiple bookmark files
:PROPERTIES:
:CUSTOM_ID: multiple-bookmark-files
:END:

A bookmark /file/ is a physical file on disk containing complete
bookmark records.  Bookmark+ makes it easy to:

- Switch to a different bookmark file (replacing the current
  in-memory set).
- Accumulate bookmarks by loading additional files.
- Save the current bookmarks to a different file with
  =bookmark-write= (=C-x x w=).
- Create an empty bookmark file with =bmkp-empty-file= (=C-x x 0=).

Switching:

| Command                                          | Key              | Meaning                                  |
|--------------------------------------------------+------------------+------------------------------------------|
| =bmkp-switch-bookmark-file-create=                 | =C-x x L= / =L=    | Switch to another bookmark file.          |
| =bookmark-load= (no prefix)                        | =C-x x l=          | Add bookmarks from a file to the current set. |
| =bookmark-load= (with prefix)                      | =C-u C-x x l=      | Equivalent to =L= (switch).               |
| =bmkp-switch-to-bookmark-file-this-file/buffer=    | =C-x x C-l=        | Switch to a file containing only this buffer's bookmarks. |
| =bmkp-bmenu-load-marked-bookmark-file-bookmarks=   | =M-l=              | Load every marked bookmark-file bookmark. |

Copying / moving / creating bookmark files from the marked
bookmarks:

| Command                                              | Key       | Meaning                                  |
|------------------------------------------------------+-----------+------------------------------------------|
| =bmkp-bmenu-copy-marked-to-bookmark-file=              | =Y > +=    | Copy marked into another bookmark file.  |
| =bmkp-bmenu-move-marked-to-bookmark-file=              | =Y > -=    | Move them.                               |
| =bmkp-bmenu-create-bookmark-file-from-marked=          | =Y > 0=    | Make a new file containing them (with prefix arg also create a bookmark-file bookmark for it). |

Persistent bookmark-file bookmarks --- created with
=bmkp-set-bookmark-file-bookmark= (=C-x x y= / =C-x x c y=) lets
you "jump to" (= load) a particular bookmark file by name.
=C-u= as a prefix argument switches to that file rather than
accumulating it.

The /default/ bookmark file for the next Emacs session is the one
in use when you quit, controlled by =bmkp-last-as-first-bookmark-file=.
Set this to =nil= to always start from =bookmark-default-file=
instead.

Note that bookmark files are /physical/, not logical, groupings.
Copying a bookmark from one to another duplicates it; the two
copies are independent thereafter.  To organize a single bookmark
across files, move it (do not copy).

* Autonamed bookmarks
:PROPERTIES:
:CUSTOM_ID: autonamed
:END:

An /autonamed/ bookmark is a regular, persistent bookmark whose
name is generated automatically from a buffer name + position
(default form: ="000058356 foo.el"= --- nine-digit position then
buffer name).  It records no region, only a position.

- =bmkp-toggle-autonamed-bookmark-set/delete= (=C-x x RET=)
  toggles the autonamed bookmark at point.  Setting one is one
  keystroke; deleting it is the same keystroke.
- With a prefix argument, =bmkp-toggle-autonamed-bookmark-set/delete=
  (=C-x x RET=) deletes /all/ autonamed bookmarks in the current
  buffer (after confirmation).
- =bmkp-delete-bookmarks= (=C-x x delete=) deletes individual
  bookmarks under cursor or all bookmarks in the buffer (any type).

Other autonamed-set commands (not bound by default):
=bmkp-set-autonamed-bookmark-at-line=,
=bmkp-set-autonamed-regexp-buffer= (one bookmark per regexp match
in buffer), =bmkp-set-autonamed-regexp-region= (same, in region),
=bmkp-occur-create-autonamed-bookmarks= (=C-c C-M-B= in =*Occur*=).

If you do not want autonamed bookmarks to persist across sessions:

#+begin_src emacs-lisp
(add-hook 'kill-buffer-hook 'bmkp-delete-autonamed-this-buffer-no-confirm)
(add-hook 'kill-emacs-hook  'bmkp-delete-autonamed-no-confirm)
#+end_src

(Alternative: make autonamed bookmarks /temporary/ via
=bmkp-autotemp-bookmark-predicates= --- the default --- and they
will not be saved.  See [[#temporary][Temporary bookmarks]].)

Customize the name format with =bmkp-autoname-format= and
=bmkp-autoname-bookmark-function=.  The special spec =%B= refers to
the buffer name within the format (=%s= works too if the name is
unambiguous).

* Temporary bookmarks
:PROPERTIES:
:CUSTOM_ID: temporary
:END:

A /temporary/ bookmark is one that will not be written to the
bookmark file.  It exists only for the current session (unless you
explicitly save it).

** Temporary-bookmarking mode

=bmkp-temporary-bookmarking-mode= (=M-L= in =*Bookmark List*=, or
=M-x bmkp-temporary-bookmarking-mode=) turns on a minor mode that:

1. Switches to an empty bookmark file.
2. Turns off auto-saving (=bookmark-save-flag=).

Every bookmark you create in this mode is therefore session-local.
The mode-line indicator changes to =TEMPORARY Bookmarking=.

** Per-bookmark temporary flag

Even with normal bookmarking, you can mark individual bookmarks
temporary:

| Command                                      | Key            | Meaning                                  |
|----------------------------------------------+----------------+------------------------------------------|
| =bmkp-toggle-temporary-bookmark=               | =C-M-X=         | Toggle for this line's bookmark.          |
| =bmkp-bmenu-toggle-marked-temporary/savable=   | =M-X=           | Toggle for the marked bookmarks.          |
| Temporary mark/show commands                  | =X M= / =X S=   | Mark / show only temporary bookmarks.     |
| =bmkp-toggle-autotemp-on-set=                  | =C-x x x=       | Toggle: make every newly set bookmark temporary. |

Outside the display, =bmkp-make-bookmark-temporary= and
=bmkp-make-bookmark-savable= flip the flag for a named bookmark.

** Autotemp predicates

=bmkp-autotemp-bookmark-predicates= is a list of predicates ---
typically type predicates such as =bmkp-autonamed-bookmark-p= ---
that decide which bookmarks become temporary automatically when
created or updated.  The default makes autonamed bookmarks
temporary.

=bmkp-delete-all-temporary-bookmarks= removes them all from the
current list at once.

Temporary bookmarks are flagged with =X= in =*Bookmark List*=
(replacing the annotation flag).

* Automatic bookmarking
:PROPERTIES:
:CUSTOM_ID: automatic
:END:

** On a hook

Add =bmkp-set-automatic-bookmark= to any hook to create a
bookmark whenever that hook runs.  Example:

#+begin_src emacs-lisp
(add-hook 'imenu-after-jump-hook #'bmkp-set-automatic-bookmark)
#+end_src

The bookmark created is an autonamed bookmark.

** Idle-period bookmarking

=bmkp-automatic-bookmark-mode= is a /local/ minor mode that sets
an autonamed bookmark whenever Emacs has been idle for
=bmkp-automatic-bookmark-mode-delay= seconds in the current
buffer.  =bmkp-global-automatic-bookmark-mode= turns it on
everywhere (a bookmark is still only created for the currently
selected buffer).

Tuning options:

- =bmkp-automatic-bookmark-mode-lighter= --- mode-line lighter
  (default =Auto-Bmk=; set to ="" to hide).
- =bmkp-automatic-bookmark-min-distance= --- minimum distance (in
  characters) from the nearest existing automatic bookmark.  =nil=
  removes the limit (still at most one per position).
- =bmkp-automatic-bookmark-set-function= --- which command to use
  (default =bmkp-set-autonamed-bookmark-at-line=).

To highlight automatic bookmarks, set =bmkp-auto-light-when-set=
(e.g. to =autonamed-bookmark=).

** Automatic Info bookmarking

Enable global minor mode =bmkp-info-auto-bookmark-mode= and every
Info node you visit either creates or updates a bookmark named
=(MANUAL) NODE=.  =bmkp-info-auto-type= chooses between
=update-only= (default) and =create-or-replace=.
=bmkp-toggle-info-auto-type= flips it.

This gives you a persistent "what I have read recently in the
Elisp / Emacs / ... manuals" log.  Sort by =s d= (last access) or
=s v= (visit count) in =*Bookmark List*=.

* Bookmark links
:PROPERTIES:
:CUSTOM_ID: bookmark-links
:END:

=bmkp-insert-bookmark-link= inserts a text-property link in any
buffer that, when you hit =RET= or click =mouse-2=, jumps to the

* Use with Icicles
:PROPERTIES:
:CUSTOM_ID: icicles
:END:

If you load /Icicles/, Bookmark+ gains:

- *Bookmark browsing.*  Jump commands become multi-commands ---
  you can act on / visit several bookmarks during one invocation,
  cycle through candidates, etc.
- *Progressive completion.*  Narrow candidates against multiple
  patterns; intersect with bookmark types (=C-M-i= Info,
  =C-M-@= remote, =C-M-r= region, ...).
- *Single key to set or jump.*  Whatever you bind
  =bmkp-bookmark-set-confirm-overwrite= to (typically =C-x r m=)
  becomes contextual: =M--= prefix jumps, numeric prefix sets
  silently using buffer + region/line text as the name, =C-u=
  prompts without overwriting.
- *=*Completions*= is a dynamic =*Bookmark List*=.*  Multi-completion
  candidates have three parts (name, file/buffer, tags); =C-,=
  cycles Bookmark+ sort orders; =C-M-RET= shows full bookmark
  info; =S-delete= deletes a candidate.
- *File-name completion gets bookmark hooks.*  =C-x m= jumps to a
  file bookmark, =C-x a a= creates an autofile, =C-x a += / =C-x a -=
  add/remove tags.
- *Icicles search-hits bookmarks.*  During Icicles search, =C-x
  C-M->= saves the current set of search-hit candidates as a
  bookmark; =C-x C-M-<= restores them; =C-x C-<= adds to them.

There are 25 Icicles commands for searching bookmark destinations.
The most general is =icicle-search-bookmark=; others filter by
type or by tag combinations.

* Compatibility with vanilla =bookmark.el=
:PROPERTIES:
:CUSTOM_ID: compatibility
:END:

Bookmark+ is compatible with vanilla Emacs 20--26+.  Bookmarks
created by either implementation can be used by the other, with
one important caveat:

=bmkp-propertize-bookmark-names-flag= (non-nil by default on Emacs
21+) makes Bookmark+ store bookmark records as /circular/ Lisp
structures (the name has the full record as a text property).
That layout is readable by vanilla Emacs.  But if vanilla Emacs
then /saves/ the file, the resulting file is unreadable --- vanilla
does not bind =print-circle= when writing.

Rule of thumb:

- If you mainly use Bookmark+ and only occasionally jump (never
  save) using vanilla Emacs, leave the flag at its default.  You
  get the niceties of multiple bookmarks with the same name (vital
  for autofiles).
- If you regularly go back and forth, /set the flag to =nil=/.
  You lose same-name bookmarks but the file stays interoperable.

To convert an existing propertized file for use with vanilla
Emacs:

#+begin_example
M-x set-variable RET bmkp-propertize-bookmark-names-flag RET nil RET
C-x r l         ; or C-x x e --- open *Bookmark List*
g               ; refresh
S               ; save
M-x bmkp-save-menu-list-state RET
#+end_example

Propertized names are also saved in any /full-snapshot/
state-restoring command (from =C-c C-S-c=).

* Library layout
:PROPERTIES:
:CUSTOM_ID: library-layout
:END:

The package is composed of the following files.  Loading
=bookmark+.el= pulls in everything required automatically.

| File              | Lines | Purpose                                                              |
|-------------------+-------+----------------------------------------------------------------------|
| =bookmark+.el=     |   ~250 | Driver: =require= the other libraries.                                |
| =bookmark+-mac.el= |   ~625 | Lisp macros (define type-specific commands, sort orders, etc.).       |
| =bookmark+-1.el=   |  ~14k | Core: bookmark types, jump handlers, set commands, tag commands, ... |
| =bookmark+-bmu.el= |   ~7k | The =*Bookmark List*= (bmenu) buffer: faces, sorting, marking, ...    |
| =bookmark+-lit.el= |   ~1.6k | Optional: highlighting of bookmark locations in their buffers.       |
| =bookmark+-key.el= |   ~1.5k | Key bindings and menu definitions.                                   |
| =bookmark+-doc.el= |   ~4.4k | Comment-only file: the canonical detailed manual.                    |
| =bookmark+-chg.el= |   ~3.9k | Comment-only file: the change log.                                   |

=bookmark+-lit.el= is optional --- if it is not on your =load-path=,
the highlighting feature is silently disabled but everything else
still works.
